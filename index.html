Faux-Par and Par-Fait

Many programs require a series of key/value to define their settings needed for their configuration and behavior. For command line programs, these key/value settings are often read from a text file. The Faux-Par classes allow for

  • the definition of settings and using the same code
  • an automatic GUI generation.

The GUI program is called Par-Fait.

Process

The C++ program needs to define its required parameters by creating FauxPar objects. In fact, each class can define its own parameters.

Then the Par-Fait GUI polls the objects FauxPar::FauxPar settings and creates the GUI for them automatically.

This way, all settings have to be defined only once, directly in the program's code. The GUI can then be updated simply by re-compiling it.

Settings Definition

A class implements the method FauxPar::FauxPar ExampleClass1::GetFauxPar(). Here's an example for a float value:

FauxPar::FauxPar ExampleClass1::GetFauxPar() {
FauxPar::FauxPar parameters("Example1");
parameters.insert(new FauxPar::ValueFloat("speed"));
return parameters;
}

In the case of the FauxPar::ValueFloat object also allows for setting allowed minimum and maximum values for the setting. Other possible setting types will comprise e.g. integers, strings, and also groups and lists of such types.

The Faux-Par library can offer different interfaces to methods on how to read and parse such settings. One possibilty is the use of the libconfig library.

GUI Generation

The Par-Fait GUI walks through the list of all objects to be configured and creates all required forms by polling the objects' FauxPar::FauxPar settings that it reads from the objects' GetFauxPar() methods.
